# lvgl项目开发指南（个人心得）

## 1.环境配置

1.需要有GCC，cmake作为底层环境变量。

2.安装vscode、guiguider、visual studio



## 2.操作流程

1.按照蓝湖给定切片图以及对应控件的坐标在guiguider上完成初步的对项目lvgl的初步开发。

//注：此阶段要按照规定的或已经完成的项目作为目标，对所设计的包括页码名称、变量、对应事件进行统一的命名、设置。便于后期对对应事件的修改以及对主机方传输数据的统一性，加快代码开发流程。

2.在guiguider上画好对应界面后,切记一定要先点击右上角的模拟器运行，以c运行模拟器后系统会自动编译之后弹出生成的模拟界面，此时在可以进行对工程的代码导入。如果未进行模拟编译直接进行工程写入对于已更改的部分不会更新到已经创建好的lvgl模拟文件中。

3.在使用guiguider绘制lvgl界面时，最好将每张图的底层页面切换逻辑在绘制阶段就设计好，对应的组件例（按键、滑动块）需要预留对应的事件触发函数，便于后期对对应的按键事件的更改。

4.在guiguider的UI界面设计完善之后，通过模拟器的认证保证页面切换的底层逻辑后即可将guiguider软件所生成的代码导入到已经准备好的vscode仿真lvgl文件中。 

## 3.vscode配置

在导入代码后使用vscode打开预备好的文件，配置好需要使用的编译器和cmake配置。

完成上述配置后将配置文件下的build\CamakeCache.txt 文件删除，再右键点击CMakelist.txt文件选择删除并重新配置所有文件。系统会按照文件路径重新对仿真生成的文件路径进行自动配置。后续对代码的修改需要重新加入新文件到仿真中是也要进行重新配置。

此时可以直接通过vscode对lvgl的代码进行自定义修改，增添guiguider软件不能添加的自定义事件。配置好之后直接在windows环境下可直接编译生成对应的仿真.exe文件。（只有第一次配置的时候需要编译全部文件之后才能生成，后续更改文件时只须编译修改文件后直接就可编译大大减少编译用时，增加开发效率）

按照上述操作之后在配置文件下的bin目录会生成main.exe文件，双击运行对应的仿真文件。后续可根据此文件对自己编译的自定义代码修改完善，知道达到预计效果。

## 4.visual studio 使用仿真验证

在对仿真文件修改到一定程度后，如果需要验证lvgl运行时所使用的空间或验证添加事件的输出可使用vs打开生成的main.exe文件，点击本地编辑运行后即可实时的观测自己所生成的文件在运行时的系统内存占比以及各事件对应的输出和可能存在的bug。

# lvgl代码总体架构

## 1.逻辑操作lvgl页面切换。

  由于lvgl所有的操作必须在主函数或者独属lvgl自己的线程中进行，目前最流行的页面控制是由lvgl内部定时器中创建消息队列接收，由接收的消息进行页面切换。
页面切换代码如下

    // 切换到开机欢迎页面的函数
    void change2LV_SP_WELCOME()                
    {
        // 获取当前活跃的屏幕对象
        lv_obj_t * act_scr = lv_scr_act();                
    // 获取当前屏幕所属的显示设备
    lv_disp_t * d = lv_obj_get_disp(act_scr);
    // 检查是否满足切换条件：
    // 1. 没有前一个屏幕
    // 2. 没有要加载的屏幕，或者要加载的屏幕就是当前屏幕
    if (d->prev_scr == NULL && (d->scr_to_load == NULL || d->scr_to_load == act_scr)) {
        // 清理当前屏幕上的所有对象
        lv_obj_clean(act_scr);
        
        // 在guiguider页面接口初始化时定义为true
        if (guider_ui.LV_SP_WELCOME_del == true) {    
            // 页面初始化
            setup_scr_LV_SP_WELCOME(&guider_ui);
        }
        // 加载到对应的界面
        lv_scr_load_anim(guider_ui.LV_SP_WELCOME, LV_SCR_LOAD_ANIM_NONE, 0, 0, true);
        guider_ui.LV_SP_WELCOME_del = true;
    }
    }

### 2.创建定时器处理消息

这样子的处理和切页逻辑处理相似，在逻辑切换点表值的同事需要再传输对应的消息更新页面的状态。类似高端麦有线款的处理。
